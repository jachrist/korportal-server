Lokal utvikling

Appen har ingen build-steg. Bruk en enkel HTTP-server:

# Alternativ 1: Python
python -m http.server 8080

# Alternativ 2: Node.js
npx serve

# Alternativ 3: VS Code
# Installer Live Server-utvidelsen, høyreklikk -> Open with Live Server
Merk: Husk å opprette js/env.js fra js/env.example.js og fylle inn Power Automate-URLene.

Deploy til Azure Static Web Apps

Fra VS Code (anbefalt)

  1. Installer utvidelsen «Azure Static Web Apps»
  2. Logg inn med Azure-kontoen
  3. Opprett Static Web App:
    • Navn: korportal
    • Region: West Europe
    • Build preset: Custom
    • App location: /
    • Output location: /
  4. Deploy oppdateringer: Høyreklikk → Deploy to Static Web App, eller koble til GitHub for automatisk deploy ved push

Med Azure CLI

# Logg inn
az login

# Opprett ressursgruppe
az group create --name korportal-rg --location westeurope

# Opprett Static Web App
az staticwebapp create \
  --name korportal \
  --resource-group korportal-rg \
  --location westeurope

# Hent deployment token
az staticwebapp secrets list \
  --name korportal \
  --resource-group korportal-rg

# Deploy med token
swa deploy ./ \
  --deployment-token <TOKEN>

staticwebapp.config.json

Konfigurasjonsfilen styrer ruting, sikkerhet og autentisering for Azure Static Web Apps.

  • routes – Ruting og rollebasert tilgang. Definerer hvilke roller som kreves for ulike stier.
  • navigationFallback – Ukjente stier reskrives til index.html (SPA-støtte). Sikrer at klient-ruting fungerer korrekt.
  • globalHeaders – Sikkerhetshoder (CSP, X-Frame-Options, etc.). Beskytter mot XSS, clickjacking og andre angrep.
  • auth – Azure AD identity provider. Konfigurerer hvilke autentiseringsleverandører som er tilgjengelige.
  • responseOverrides – Tilpassede feilsider:
    • 400 – Ugyldig forespørsel
    • 401 – Omdirigerer til innlogging
    • 403 – Ingen tilgang
    • 404 – Side ikke funnet

Deploy-sjekkliste

  1. Bump SW-cacheversjoner i sw.js hvis statiske filer er endret
  2. Sjekk at env.js har korrekte endepunkt-URLer
  3. Test lokalt med HTTP-server
  4. Deploy via VS Code eller GitHub push
  5. Verifiser at siden laster korrekt i nettleseren
  6. Sjekk DevTools → Application → Service Workers for oppdateringsstatus
