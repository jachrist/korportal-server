Innloggingsflyt

Autentisering skjer via engangskode sendt til e-post:

  1. Bruker oppgir e-postadresse pa login.html
  2. Frontend kaller POST /auth/send-kode (Power Automate sender e-post med 6-sifret kode)
  3. Bruker taster inn koden
  4. Frontend kaller POST /auth/verifiser-kode
  5. Ved suksess returneres medlemsdata (id, name, email, role, voice)
  6. Data lagres i localStorage under nokkelen korportal-member

localStorage-nokler

Nokkel Innhold Brukes av
korportal-member JSON med medlemsdata (id, name, email, role, voice) navigation.js, member-utils.js, alle sider
korportal-theme "dark" eller "light" navigation.js (ThemeManager)
korportal_badge_lastSeen_* ISO-tidsstempel for sist besok per seksjon badge-manager.js

Rollehierarki

Roller er hierarkiske. Hoyere rolle inkluderer alle lavere. Definert i ROLE_HIERARCHY i navigation.js.

admin  -> [admin, styre, medlem, anonym]
styre  -> [styre, medlem, anonym]
medlem -> [medlem, anonym]
anonym -> [anonym]

hasRole('styre', 'medlem') returnerer true — styre har tilgang til medlem-sider.

hasRole('medlem', 'styre') returnerer false — medlem har ikke tilgang til styre-sider.

Tilgangskontroll per side

Hver side kaller initPage() med requireAuth og requiredRole. Hvis autentisering feiler, omdirigeres brukeren til login.html. Hvis rollen ikke er tilstrekkelig, omdirigeres brukeren til /.

initPage({ requireAuth: true, requiredRole: 'styre' })

Navigasjonsmeny og roller

Meny-elementer fra SharePoint har minRole-felt. filterNavItemsByRole() filtrerer basert pa innlogget brukers rolle. hideWhenLoggedIn skjuler «Logg inn»-lenken for innloggede brukere. isLogout markerer utloggingsknappen.

Utlogging

handleLogout() i MenuManager: Tommer badge-data, fjerner korportal-member fra localStorage, omdirigerer til /.
